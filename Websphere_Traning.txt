2. Horizantal Cluster -

		Pre-requisites - Add the inbound port in AWS console.

		In a horizontal cluster, JVMs that reside on different physical machines are clustered. 
		
		The obvious advantage to a horizontal cluster is that if one physical server goes down, 
		the other physical server can continue to produce the applications in the JVM, resulting in high-availability. 
		Also, from a systems engineer perspective, a horizontal cluster lets the engineer do work on one cluster member without bringing down the applications being produced by the JVMs in the cluster.
		For example, the engineer could bring down the server01 JVM, patch the server, restart the JVM, and bring it back up, without causing an outage, as the server02 JVM can still produce the application.

		1. Transction manager form Application server will ask/ check the resources (Dabase sever) whether they are ready to commit or not. 
   
   - To maintain the connection pool we are having pool manager therad thi is also maintain by on f th conneciton pool properties. (reap time) by defualt 180 sec.

	Type1 - JDBC , ODBC
	Type2 - Thick Driver 
	Type3 - Network driver
	Type4 - Think driver - with java.
	
	1. Entire pool
    2. Failing connection 
Properties and prameters : 

	1. Maximum conneciton : - Specifies the max number of connection / you can create. 10 connect.
	2. Min conneciton :- specifies the minimum number of connection. (0)
	3. Reap time : the interval in second between runs of pool maintainer thread. 
	4. Connection Time out : Number of sceconds a request for a connection waits , when there are not coonection available in pool. ( 180 ssec)
    5. unused timeout : The interval in seconds after an idle or unused connection is disconnection. (1800 sec , 30min)
	6. Aged Time : Specifies the interval in seconds before physical conneciton is discarded. [0]
	 
Purge Policy : - How to purge conneciton when a stale conneciton or fatal conneciton error is detected. 
	
  1. Entire pool - it will discarded all connection 
  2. Failing connection - only failed connection will discard.

🟣 DEBUG+	Very detailed	*=finer
🟣 TRACE	Maximum detail	*=finest
